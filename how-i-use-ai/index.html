General Tips (Any AI Tool)

Dictation with Wispr Flow

Speaking is much faster than typing. If you spend a lot of time writing prompts, consider using a speech-to-text tool. I use Wispr Flow, which transcribes your speech in real time. It dramatically speeds up my interaction with AI—I can dictate a paragraph-long prompt in seconds instead of typing it out.

Handling accent and transcription errors. If you are not a native English speaker, any speech-to-text tool will occasionally mishear words, drop articles, or produce run-on sentences. In Claude Code, I solve this with a hook—a small script that runs automatically every time you submit a prompt. The hook injects a notice telling Claude that the input was dictated and may contain transcription artifacts, so Claude silently corrects them instead of asking for clarification.

Here is the hook script I use. Save it as ~/.claude/hooks/dictation-context.sh:

#!/bin/bash
cat <<'EOF'
DICTATION NOTICE: The user's prompt was likely dictated
using Wispr Flow (speech-to-text). The user is a native
Chinese (Mandarin) speaker. The transcription may contain:
- Homophones or near-homophones (e.g. "EA" → "it")
- Misheard technical terms or proper nouns
- Missing or wrong articles, prepositions, or plurals
- Run-on sentences or missing punctuation

Mentally correct any likely transcription artifacts and
interpret the intended meaning. Do not ask for clarification
on obvious transcription errors — just fix them silently.
EOF

Then activate it by adding this to your ~/.claude/settings.json:

{
  "hooks": {
    "UserPromptSubmit": [
      {
        "type": "command",
        "command": "$HOME/.claude/hooks/dictation-context.sh"
      }
    ]
  }
}

Adapt the script to your own native language—just change "Chinese (Mandarin)" and the example error patterns to match your accent.

Claude Code

Claude Code is a terminal-based AI assistant that can read your files, write code, run commands, and interact with your project directly. Watch this video by Paul Goldsmith-Pinkham for a quick introduction, or see the official documentation for installation and a full overview.

Getting Started

My setup: VS Code + Terminal CLI

I use Claude Code through the terminal CLI inside VS Code. This combination gives me the best of both worlds:

  • VS Code lets me easily see the project folder structure in the sidebar, quickly open and inspect files, and track which files Claude creates or modifies.
  • The terminal CLI (run claude in VS Code's integrated terminal) is where I actually interact with Claude Code. The CLI uses significantly fewer tokens than the VS Code extension or the desktop app for the same work, which matters for long sessions.

To set this up: open your project folder in VS Code, then open the integrated terminal (Ctrl+`) and type claude.

First thing to do: run /init

The first thing to do in any new project is run /init inside Claude Code. This generates a CLAUDE.md file—a plain-text document where you describe your project, your preferences, and any instructions you want the AI to follow every time it works in that directory.

Think of it as a briefing sheet. Without it, you re-explain context at the start of every session. With it, Claude Code already understands your setup.

Example of what a CLAUDE.md might look like:

# Project: My Research Paper

## What this project is
Analysis of WWII draft records using Stata and R.

## My preferences
- Use Stata for data cleaning, R for figures
- Commit changes with descriptive messages
- Ask before deleting files
- I use dictation, so prompts may be conversational

Basic Tricks

Treat Claude Code as an RA for your entire life

Claude Code has high execution power (it reads files, writes code, runs commands) and high intelligence (it reasons, plans, adapts). That combination makes it a research assistant that can handle almost any task you throw at it—not just coding, but admin, writing, data work, travel logistics, anything.

The key insight: even if the first result is mediocre, it is worth teaching Claude Code how to do the task well. The investment pays off permanently. Every skill you build, every memory you save, every line in your CLAUDE.md makes every future session better—across every project, on every machine, forever.

Not sure how to teach it? Just ask:

I want Claude Code to do [X] better in the future.
What's the best way to achieve this — a skill,
a hook, a memory, a CLAUDE.md entry, or something else?

Claude Code will tell you which mechanism fits your case. You don't need to know the system upfront—just describe what you want, and let it guide you to the right persistence layer.

Plan mode for complex tasks

For any task that involves more than a few steps, start in plan mode (press Shift+Tab to toggle it on). In plan mode, Claude Code thinks through the problem and proposes a structured plan before making any changes. You review and approve the plan first, then it executes.

This prevents the AI from charging ahead in the wrong direction. It is especially useful for multi-file refactors, setting up a new project structure, or any task where the order of steps matters.

Example prompt:

I want to restructure this Stata project so that
each .do file handles one step of the pipeline
(clean, merge, analyze, figures). Plan this out first.
The reflection workflow

This is perhaps the most powerful technique I have found. When you encounter a new type of task for the first time:

  1. Push Claude Code to its best. Work through the task iteratively. Keep refining.
  2. Do not delete the session history. Even if it costs tokens, the full history is essential for what comes next.
  3. Ask Claude Code to reflect. Say something like:
Reflect on this entire session. Is there anything you
should save as a skill, a memory, or a setting change
so that next time we do this, it goes faster?

Claude Code will then create reusable artifacts—skills (saved workflows), memory entries (facts to remember), or configuration changes. The first time you do a task with AI might not save time. The second time, it will.

For more on creating skills, see this walkthrough.

Validation of complicated tasks

The agent that writes a plan has planner bias—like an author proofreading their own paper. It rationalizes gaps rather than catching them. The same applies after execution: the agent that built the pipeline is the least likely to notice what it missed.

The solution: two automatic validation gates.

  • validate-plan — Before execution begins, dispatches a fresh subagent (no shared context) to review the plan across 7 dimensions: data integrity, statistical logic, pipeline completeness, replication feasibility, scope, error handling, and sequencing. Verdict: PROCEED, REVISE FIRST, or STOP AND DISCUSS.
  • validate-execution — After execution finishes, dispatches a fresh subagent to audit the completed work across 6 dimensions: plan compliance, output verification, code logic, replication documentation, data integrity, and completeness. Verdict: COMPLETE, NEEDS FIXES, or NEEDS DISCUSSION.

Because each reviewer starts with a blank slate, it sees the work cold—the way a referee or replication auditor would. The key principle: don’t let the same agent that creates a plan be the only judge of that plan.

Install the skills. Download both skill files:

# If you use the Dropbox sync setup:
mkdir -p ~/Dropbox/AI_meta/claude-sync/skills/validate-plan && \
curl -o ~/Dropbox/AI_meta/claude-sync/skills/validate-plan/SKILL.md \
  https://gjiang-economics.github.io/skills/validate-plan.md

mkdir -p ~/Dropbox/AI_meta/claude-sync/skills/validate-execution && \
curl -o ~/Dropbox/AI_meta/claude-sync/skills/validate-execution/SKILL.md \
  https://gjiang-economics.github.io/skills/validate-execution.md

# Otherwise:
mkdir -p ~/.claude/skills/validate-plan && \
curl -o ~/.claude/skills/validate-plan/SKILL.md \
  https://gjiang-economics.github.io/skills/validate-plan.md

mkdir -p ~/.claude/skills/validate-execution && \
curl -o ~/.claude/skills/validate-execution/SKILL.md \
  https://gjiang-economics.github.io/skills/validate-execution.md

Add to your CLAUDE.md. Paste this so the skills trigger automatically for research projects:

## Automatic Validation for Research Projects

For **research project work** (data analysis, pipeline construction,
multi-file empirical code in Stata/R/Python), two validation gates
are mandatory:

### Pre-execution: `validate-plan`
After creating a multi-step plan for a research project, ALWAYS
invoke the `validate-plan` skill before beginning execution.

### Post-execution: `validate-execution`
After completing execution of a multi-step research plan, ALWAYS
invoke the `validate-execution` skill before claiming the work
is done.

### How to determine "research project work"
A task qualifies if ANY of these are true:
- Plan involves data files (.dta, .csv, .parquet, .rds)
- Plan involves running Stata .do files, R scripts, or Python data scripts
- Plan involves regression, estimation, or statistical analysis
- Plan has 3+ steps that modify code or data files

Then invoke manually anytime with /validate-plan or /validate-execution. You can view the full skill files here: validate-plan, validate-execution.

Cross-model brainstorming

Sometimes Claude Code struggles with a specific subtask—perhaps a tricky mathematical derivation or an unusual data transformation. When this happens after a few attempts, try getting a "second opinion" from another model:

  1. Ask Claude Code to summarize the situation: what the problem is, what has been tried, and what is not working.
  2. Say: "I want to brainstorm this with another model. Give me a self-contained prompt I can paste into Gemini."
  3. Copy the prompt into Gemini (or another AI you trust for that type of problem).
  4. Paste the solution back into Claude Code and ask it to integrate it.

Different models have different strengths. Combining them often gets you to a solution faster than pushing a single model repeatedly.

Remote control

Claude usage is calculated in 5-hour windows. If Claude Code pauses mid-task waiting for a permission confirmation and you are away from your desk, those hours are wasted. Remote Control solves this: it lets you monitor and approve Claude Code sessions from your phone or any browser, so you can keep tasks moving even when you are not at your computer.

Remote Control connects claude.ai/code or the Claude mobile app (iOS, Android) to your local Claude Code session. Claude keeps running on your machine the entire time—the phone or browser is just a window into that session.

How to use it. If you are already in a session, type:

/remote-control

Or start a new session with Remote Control enabled:

claude --remote-control

Visit the Claude app or claude.ai/code, where you can send messages from your phone or your browser.

My workflow. Before I leave the office, I start a long task with /remote-control. On the train home, I check my phone, approve any permission prompts, and review progress. The task keeps running without me sitting at my desk.

Chaining sessions. If I know what to do after the current task finishes, I start a second session with /remote-control before leaving. When session one completes, I pick up session two from my phone and continue—using the results from session one.

See the official documentation for full details on server mode, security, and troubleshooting.

Syncing Claude Code across machines

I use Claude Code on two machines: a MacBook and a Windows desktop at the university. To keep the setup identical, I store all shared config in a Dropbox folder and create symbolic links from ~/.claude/ on each machine.

What gets synced (via Dropbox):

Dropbox/AI_meta/claude-sync/
├── memory/        # Your personal AI memory
├── skills/        # Custom skills
├── commands/      # Custom slash commands
├── plugins/       # Plugins
├── CLAUDE.md      # Global instructions
├── settings.json  # Shared preferences
└── .mcp.json      # MCP server config

What stays local: settings.local.json, cache, history, and other runtime files.

On Mac, create symlinks:

ln -sf ~/Dropbox/AI_meta/claude-sync/memory   ~/.claude/memory
ln -sf ~/Dropbox/AI_meta/claude-sync/skills   ~/.claude/skills
# ... same for each item

On Windows (PowerShell as Administrator):

# Folders — use Junction
New-Item -ItemType Junction -Path "$env:USERPROFILE\.claude\memory" `
  -Target "C:\Users\YOU\Dropbox\AI_meta\claude-sync\memory"

# Files — use SymbolicLink
New-Item -ItemType SymbolicLink -Path "$env:USERPROFILE\.claude\CLAUDE.md" `
  -Target "C:\Users\YOU\Dropbox\AI_meta\claude-sync\CLAUDE.md"

Any change on one machine instantly appears on the other. Machine-specific settings (like the path to Stata) go in settings.local.json and do not interfere.

Claude Code for Administrative Work

Travel reimbursement

After a conference or workshop, I use Claude Code to prepare all the reimbursement documents. I have built a skill for this—a saved workflow that Claude Code follows step by step. Below is how it works and how you can set it up yourself.

Step 1: Organize your folder. Create a folder for the trip with this structure:

reimbursement/
├── proofs/                        # Receipts and payment evidence
│   ├── 01 flight receipt.pdf
│   ├── 01 boarding pass.pdf
│   ├── 03 hotel invoice.pdf
│   ├── 06 taxi receipt.pdf
│   ├── 11 city tax photo.JPG      # Photos of paper receipts work too
│   └── ...
├── host_reimbursement_policy.pdf   # Instructions from the host
├── summary_template.xlsx           # Excel template from the host (if any)
└── self_declaration_form.pdf       # Any other forms from the host

The key convention: files in proofs/ are numbered by item. The leading digits before the first space form the item number. Files sharing the same prefix (e.g., 01) belong to the same expense.

Step 2: Install the skill. Run one of these commands in your terminal:

# If you use the Dropbox sync setup (see above):
mkdir -p ~/Dropbox/AI_meta/claude-sync/skills/reimburse-external && \
curl -o ~/Dropbox/AI_meta/claude-sync/skills/reimburse-external/SKILL.md \
  https://gjiang-economics.github.io/skills/reimburse-external.md

# Otherwise, install directly:
mkdir -p ~/.claude/skills/reimburse-external && \
curl -o ~/.claude/skills/reimburse-external/SKILL.md \
  https://gjiang-economics.github.io/skills/reimburse-external.md
Click to see what the skill contains

The skill walks Claude Code through a 10-phase workflow: read the host's reimbursement policy, catalog all receipts by item number, verify each receipt against the host's rules, calculate the total claim with currency conversion, fill in the Excel template, merge all receipts into a single PDF, verify the merged PDF, and draft the submission email. You can view the full skill file here.

Step 3: Run it. Open the reimbursement folder in VS Code, start Claude Code in the terminal, and type:

/reimburse-external

Claude Code will follow the skill automatically: read the host's policy, catalog your receipts, verify everything matches, fill in the Excel template, merge receipts into a single PDF, and draft the submission email.

Adapting this for your own use. This skill is designed for external reimbursement (conference hosts, funding bodies). If your university has its own reimbursement process, create a separate skill for it. The first time you do a reimbursement with Claude Code, ask it to reflect at the end and create a skill tailored to your institution's specific forms and policies.

Learning in an AI-Driven Economy

Use AI to design a self-guided study plan

Pick a topic you want to learn. Tell the AI your background, how much time you have, and what you want to understand. Ask it to build a study plan—readings, key ideas, and an order to work through them. As you read, use the AI to ask questions, check your understanding, and explore connections. What used to take days of searching and filtering now takes hours.

This idea comes from a post by Jesús Fernández-Villaverde, who used AI to design a self-guided study of sociologist Erving Goffman—and found the experience comparable to a master's-level course week.
